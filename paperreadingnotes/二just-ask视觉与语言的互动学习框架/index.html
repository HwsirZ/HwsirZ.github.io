（二）Just Ask：视觉与语言的互动学习框架

Publish on: 2025/11/18
Words: 1795
Just Ask: An Interactive Learning Framework for Vision and Language Navigation

一：论文信息

  • 标题：Just Ask: An Interactive Learning Framework for Vision and Language Navigation

  • 作者：Ta-Chung Chi, Mihail Eric, Seokhwan Kim, Minmin Shen, Dilek Hakkani-Tur

  • 出处：The Thirty-Fourth AAAI Conference on Artificial Intelligence (AAAI-20)

  • 年份：2020

  • 链接： (https://doi.https://doi.org/10.1609)

二：背景介绍

对于目标导航任务中出现的模糊歧义指令，或者在行进过程中遇到模糊场景，机器人无法做出有效决策时，通过向人类提问，来进一步明确下一步动作的执行。

要解决的问题

如何在遇到歧义指令或者复杂环境时，和用户进行交流获得指导，同时避免频繁的打扰用户

三：工作内容

  1. Instruction_Based Navigation，基于指令的导航

  2. Human Robot Interaction，人机交互

  3. Active Learning，主动学习

1. Instruction_Based Navigation

  基于指令的目标导航在VLN任务中有广泛的应用，但是已有研究缺少与人的交互，因此对于复杂/模糊/有歧义的环境，往往会失败。

2. Human Robot Interaction

  对于交互，已有工作是在任务开始之前进行交互，而任务中也有进行交互的必要性，与人类之间的交互不同，人机之间的交互，或者更明确的说，机器向人类进行提问时，应基于自身对环境建模的歧义程度，考虑提问时机、提问内容，以减少人机交互损耗，同时保证导航成功率。

3. Active Learning

Active learning 基本概念

传统的监督学习是收集大量数据，通过人工对数据进行标注以用于训练

在有大量数据未标注，同时标注成本高昂的情况下，模型主动选择有用的样本进行训练。具体的流程的就是模型先在现有已标注数据集上进行训练，然后通过一定的策略机制选择未标注数据中对于训练有价值的数据进行训练，这个过程中，对于挑选的未标注数据，需要督导（专家）进行人工标注。

基本架构

两个基本准则

  1. 不确定性准则

  2. 差异性准则

不确定性越大表示数据包含的信息越多，差异性越大表示样本更能全面的代表总体数据

半监督学习/直推学习/主动学习的区别

  半监督学习不需要人工干预，直推学习只关注当前未标记的数据集的标签，不考虑通用性以及对未来未知数据的泛化，主动学习主动检索未标记数据中有用或者相对最有用的数据。

4. Data Augmentation

在原有指令-轨迹数据集的基础上，通过数据增强方法，生成更多的训练样本，比如保持轨迹不变，给一条轨迹配备不同的语言指令；利用无监督学习+Speaker Model，让智能体先随机走一段轨迹，然后Speaker Model根据轨迹生成语言指令。上述方法需要海量的数据，并且生成的指令由机器生成，不具有人类的语言特性，难以在现实中使用。这里的数据增强方法需要人类参与，即少量的提问+人类高质量的回答。

三：问题建模

  • 场景： Room-to-Room (R2R) 视觉-语言导航

  • 指令： 一个有 $l $ 个词的指令$ ，I= { w_1,w_2, \dots , w_i }1^{l}$ ，其中$w_i$是word token，指令描述了一条导航路径$R = { v_1, v_2, \dots v_j }{j=1}^n$

  • 建模： 将问题转化为一个MDP问题，时间步$t = 1, 2, \dots$，状态$S_t$ ，动作$A = {a_k^t}_{k=1}^m$，状态被表示为$360^。$的全景图，具体为36个视角的全景，每个视角包括一个RGB图像、水平角度、垂直角度

  • 特征抽取： 利用ResNet进行特征抽取:

$$ f_i^t = { {Res(p_i^t), \sin \theta_i^t, \cos \theta_i^t, \sin \phi_i^t, \cos \phi_i^t }, i = 1, \dots, 36} $$

Tips

Token的概念

NLP中对一个自然语言进行划分，word/subword token，对句子中的词进行划分，在大模型中，token可能是一个词、一部分词、甚至中英文混合的一小段字符，也就是对自然语言进行tokenization（分词）的结果

四：模型

指令的词向量表示

$$ \vec{w} = Embedding(w_i) \tag{1} $$

双向LSTM解析

$$ { u_i }{i=1}^l = LSTM{ \vec{w}{i=1}^l } \tag{2} $$

$u_i$ 是正向和反向输出的串接

全景视图

$$ \vec{f}i^t = \sum{i=1}^{36} \alpha_i^t f_i^t \tag{3} $$

权重计算

$$ \alpha_i^t = softmax_i(f_i^t W_f \vec{h}^{t-1}) \tag{4} $$

$\vec{h}^{t-1}$ 是原指令感知的隐藏状态，$W_f$ 是一个可学习矩阵

解码器

自回归LSTM，每个时间步的输入是前一个动作和注意力视觉特征的串接

$$ h^t = LSTM([\vec{f}^t;\alpha^{t-1}],\vec{h}^{t-1}) \tag{5} $$

专注于指令的正确部分

$$ \vec{u}^t = \sum_{i}^{t} \beta_i^t u_i \tag{6} $$

基于所有指令词汇计算权重

$$ \beta_i^t = softmax_i(u_i W_u h^t) \tag{7} $$

$W_u$ 是一个可学习矩阵，隐藏状态 $h^t$ 的计算为:

$$ \vec{h}^t = tanh(W_h(\vec{u}^t;h^t) \tag{8} $$

计算当前时间步的动作分布

$$ P^t({ a_k^t }_{k=1}^m) = softmax_k(g_k^t W_a\vec{h}^t) \tag{9} $$

*优化

当智能体在时间步t到达(viewpoint)$v_t$时，环境会计算$\vec{R}t = { V_t, V{t+1} \dots V_n}$ ，这是从$v_t$到$v_n$的最短路径，一系列动作被称为“教师动作”，用于提供监督信号。使用交叉熵作为教师动作与$P_t$(智能体预测的动作概率分布)作为监督学习损失。

对于强化学习，智能体根据公式(9)计算出动作分布$P_t$来采样其实际动作，使用优势演员-评论家（Advantage Actor Critic, A2C）作为强化学习算法
