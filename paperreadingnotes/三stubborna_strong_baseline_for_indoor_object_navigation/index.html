（三）Stubborn：A_Strong_Baseline_for_Indoor_Object_Navigation

Publish on: 2025/12/02
Words: 4675
Stubborn是一个为Habitat ObjectNav挑战设计地baseline,策略简单但是十分有效

论文信息

Title: Stubborn: A Strong Baseline for Indoor Object Navigation Abstract: We present a strong baseline that surpasses the performance of previously published methods on the Habitat Challenge task of navigating to a target object in indoor environments. Our method is motivated from primary failure modes of prior state-of-the-art: poor exploration, inaccurate object identification, and agent getting trapped due to imprecise map construction. We make three contributions to mitigate these issues: (i) First, we show that existing map-based methods fail to effectively use semantic clues for exploration. We present a semantic-agnostic exploration strategy (called STUBBORN) without any learning that surprisingly outperforms prior work. (ii) We propose a strategy for integrating temporal information to improve object identification. (iii) Lastly, due to inaccurate depth observation the agent often gets trapped in small regions. We develop a multi-scale collision map for obstacle identification that mitigates this issue. public Time: 2022年 WebSite: github webSite Doi: arXiv

这篇论文基于以往研究中出现的问题，弱探索、目标识别准确度低、具身代理宕机等问题，提出了一个与语义无关的探索策略，同时提出一种整合时间信息以改进对象识别的策略，新增用于障碍物识别的多尺度碰撞图，环境代理困于某一区域的问题。

  1. 语义指语义地图上表明的物体类别等信息，探索目标时，可以根据语义信息之间的相关的性，有限探索相关区域，无语义策略就是指不利用现有的语义信息，类似盲目搜索
  2. 为什么说已有方法没有有效利用语义，作者通过对比实验说明了这个问题，已有研究SemExp在有语义和无语义上差别不明显，约**2~3%**左右

  1. 基于时间信息改进对象识别：以前的做法都是逐帧独立分类，每一步用当前帧的语义分割结果判断是否有目标类别，误判或者步数超限就失败；而Stubborn设计两个阶段的检测流程，先是普通帧检测，设置一个置信阈值，再走进了判断
  2. 在habitat仿真环境中，agent可能会一直原地撞墙/撞家具、或者在一个很小的空间里来回转圈、反复尝试相同路径，这种情况往往由于深度测量不佳、场景网格(mesh，划定了agent的活动空间)本身有瑕疵/碎片，位姿和地图是离散栅格，存在对齐误差。本文在深度栅格占据图的基础上，又维护了两张由“真实碰撞”生成的栅格图，一张为pessimistic collision map，另一张为optimistic collision map，即每次碰撞时，划定的障碍物范围前者大后者小。在此基础上，trapped 发生率有所下降

一：背景介绍

室内目标导航能力对于许多机器人应用至关重要，室内目标导航的形式有很多，目标实例导航是最常见的问题类型，也是本论文关注的问题重心，在这个问题中，agent的观测只有当前视角的深度图以及位姿信息。

作者将目标实例导航（Object Navigation）与点导航（Point agent需要有识别物体的能力，利用场景语义信息加速探索（如果目标是冰箱，那么就无需探索浴室,如果目标是一张床，那就直奔卧室）。

作者对比了2020年与2021年的冠军之作–（SemExp and EEAux），现有语义分割模型的精度很高，达到了90%，直观上这不会在很大程度上影响目标导航任务，但是实际上即使是90%的准确率也是不够充分的，如果agent走100步，那么前99步没有发生错误的概率就是$(90%)^{99} = 2.95% $,作者通过实验观测也指出SemExp与EEAux失败的主要原因也是检测错误

基于上述分析，作者提出了STUBBORN agent 主要贡献

  1. 通过累计每帧的检测得分改进目标检测
  2. 鉴于SemExp对于语义利用的低效，提出一种更简单的无语义探索策略
  3. 通过碰撞图的多尺度表示避免agent受困

二：实验设置

Object Navigation要求agent在一个未知环境中导航到指定的目标物体附近，输入时观测到的RGB-D图像以及位姿信息，在每一个时间步，有四个动作空间{forward, turn left, turn right, stop}，时间步不超过500.

基于Habitat Challenge的标准评测，使用MatterPort3D数据集上的2000个episodes评估模型性能。

两个评估指标

  1. Success Rate: 成功率

  2. Success weighted by inverse Path Length(SPL): 加权平均成功路径长度

    • $ \frac{1}{N} \sum_{i=1}^{N} S_i \frac{l_i}{max(p_i, l_i)}$

N是episode数量，$S_i$是成功的指示变量（成功为1失败为1），$l_i$是agent与目标之间的最短路径，$p_i$是agent的实际路径长度

三：2020与2021年冠军模型分析

SemExp(2020)与EEAux(2021)使用的是完全不同的方法,SemExp显示构建环境图，EEAux是一个端到端的模型，使用循环神经网络来实现空间记忆，分别代表了有图与无图的最新技术（2022年）。

模型错误分析

  1. 目标检测引发的问题
  2. 探索或者其他原因引发的问题

更具体一点

  • False Detection: 错误的检测
  • Missed Detection: 没能检测到，即使在视野内
  • Loop: 在一个区域内循环
  • Trapped: 受困
  • Explore: 探索错误
  • Stairs: 目标位于不同楼层而无法探索
  • Misc：其他故障

SemExp(2020)

作者基于公开发布的模型，通过200个validation episodes，对6个对象类别做了测试（Habitat使用的为21个，但是模型只支持6个）。

大约$48%$的错误是由于目标检测错误引起的，作者通过实验评估了这种成功率的提升是因为检测错误的改善引起的，而不是由于语义的利用，如 图一 所示。

EEAux(2021)

EEAux与EEAux GT（ground truth）的数据显示，SR有很大的提升，这证明SR的提升来源于分割准确率的提升。

四：Method

作者提出了一个名为Stubborn的模块化架构，如下图所示

Stubborn由四部分组成

  1. 建图模块：一个俯视的2D栅格图
  2. 探索器：将2D栅格图作为输入，输出2D网格坐标指示目标
  3. 路径规划器：输出一系列的动作指令
  4. 多帧目标检测器：检测agent是否成功抵达目标位置

Details

A. Mapping Module

代理维持一个大小为$N * N * 7（N * N = 7200 * 7200）$地图$M$,$N * N$代表像素点的多少，每个像素点25$cm^2$, 如下图所示：

7个通道中前三个是二值定义，具体含义如下图所示：

通道含义

  1. 用深度相机得到的占据图，1表示障碍物，0表示自由空间(可行走区域)
  2. 由碰撞信息得到的悲观碰撞图
  3. 由碰撞信息得到的乐观碰撞图
  4. 某个栅格在视野中出现的次数，可表示这个区域被探索的充分程度
  5. 语义分割网络对目标识别在这个栅格上的置信度累加和
  6. 历史上目标类别在此栅格的最大置信度
  7. 所有非目标类别在此栅格的最大置信度

由于深度图的不准确/mesh缺陷等问题，仅依靠这些建立占据栅格图并不充分，（笔者在想：实际上现实中，机器总会发生碰撞吗？现实中可以通过超声波等其他技术方案做避障，这种设计应用到现实中起作用还是单纯的算力与存储损耗呢）。由于代理的位置是离散化的，并且障碍物的大小未知，所以通过碰撞构建占据地图并不合适，如果障碍物被标记的尺寸与其实际尺寸不同，或大，或小，都会影响路径规划做规划，大了，路径规划器不会选择走这条路，小了，又走不过去。因此作者新增了两个通道表示，也就是悲观和乐观碰撞图，碰撞被定义为当代理想要前进，但是位置并没有改变。

悲观碰撞图将代理面前较大的区域($25 * 25 cm^2$)标记为障碍物，而乐观碰撞图将代理面前较小的区域($ 15 * 15 cm^2 $)标记为障碍物。策略： 始终将代理访问路径上的坐标标记为自由空间。这减少了代理因错误标记而被困的机会，通道2、3能够被路径规划器使用。

B. Exploration Planning Module

由于之前作者发现语义信息对目标导航探索的帮助很小，所以作者构建了一种简单的基于规则的探索方法，该方法不依赖语义信息。方法很简单就是不断地探索，直到失败为止。

将地图地前三个通道作为输入，exploration planning module输出一个以代理为中心，大小为$K * K$(K = 1200)的local Map(如下图所示)作为探索目标，这同样能减少计算量以提速

防碰撞探索 路径规划器使用深度障碍图与悲观碰撞图以减少智能体被困的机会，被困率测试如下图所示：

C. Path Planning Module

Local Map的2/3通道作为输入以及目标栅格坐标作为输入，使用快速行军法计算最短路径，路径是一个二维坐标序列，下一个坐标用于推断代理动作。路径每走一步就重新规划路径。

Choice of Obstacle Map 障碍图的选择策略，决定何时使用悲观碰撞图，何时使用乐观碰撞图

  1. 使用悲观碰撞图：路径规划器的目标是$g_{exp}, $(去地图的四个角探索的目标点)，始终使用悲观碰撞图，也就是尽可能地探索更大地区域
  2. 使用乐观碰撞图：如果目标是local map区域内，就使用乐观碰撞图做细致化的探索

暴力破解模式 自由空间非常小的时候，由于代理位置相对于map是离散化的，路径规划器可能无法找到可行路径。如果代理在十个连续步内都失败了，路径规划将进入暴力破解模式(究极进化)，交替左右转动前进，也就是挤进去。

D. Target Object Detection Module

SemExp的目标检测方法只针对单帧做检测，由于一步错，毁终身，例如如下图所示，代理在前三帧中成功识别了sofa(沙发)，但是在第四帧却将sofa错误识别维为bed(床)，导致任务失败

根据帧序列检测，作者设计了一个二阶段方法减小检测错误，检测到代理对象后，代理进入候选接近阶段，到达候选目标后，代理出发目标检测模块来决定是停止还是继续探索，如下图所示：

同SemExp一样，第一个阶段对Local Map内，代理的每个view($ V $)的每个栅格每种类别做置信预测($c_l \in [0:1]; l \in [1,L]$， L: object categories)，${ (x,y) \rightarrow [C_1^{x,y}, C_2^{x,y}, \dots , C_L^{x,y}], \forall (x,y) \in V }$。

根据EEAux，作者使用RedNet模型针对MatterPort3D的100k的随机采样做了微调，如果检测到目标物体的存在($g_{tgt}: (x,y) s.t.c_{gtg}^{x,y} > 0.5$)，那么代理就会尝试靠近物体，当代理与目标的距离达到阈值时(1m)，进入验证阶段以减少错误检测的概率

验证阶段 验证阶段使用一个二元分类器，使用手动设计的4D特征捕获目标对象的时间信息，以确定检测是否错误。4D特征向量是代理在探索环境时构建的地图的四个通道(上图已给出)

  1. Total View: $M_L[y,x,4]$
  2. Cummulative Confidence：$M_L[y,x,5]$，累积置信分数$\sum_{i=1}^{t}c_{tgt}^{x,y}(i)$
  3. Max. Confidence：$M_L[y,x,6]$，最大目标类别置信分数$max_{i \in [1,t]}c_{tgt}^{x,y}(i)$
  4. Max. Other Confidence：$M_L[y,x,7]$，最大非目标类别置信分数$max_{l \neq tgt}[c_{l}^{x,y}(i)]$

验证(系统)器是一个二元朴素贝叶斯分类器，使用基于Matterport3D验证集的数据集进行训练，防止过拟合。也就是说在验证集上做验证器，避免在训练上产生的过拟合，而导致在验证集中错误验证。每个目标类别单独训练一个二元朴素贝叶斯分类器，同时在200个时间步之前，做验证，200个之后不做验证，避免提前杀死探索，200个时间步之前，agent还有充分的时间去探索，这时候通过验证即使错判，代理后面仍有探索纠正空间，但是200个时间步之后，代理剩余时间变少，多探索，避免误判导致的失败。

五：Results

主要贡献：

  1. 相对于当时的SOTA方法，无语义探索策略的成功
  2. 多尺度碰撞图促进探索
  3. 多帧目标检测减少误报

A. Exploration Efficiency

由于成功率与目标检测以及探索的性能有关，为了独立研究探索效率，因此作者提出以下指标

  1. Ground Truth (GT) Exploration Rate：如果代理视野中目标占据了0.8%的像素，就认为物体看到了，而不是使用检测模型去判断。即这个指标只关注代理能不能移动到目标位置，而取消检测的影响，换句话说，这模拟了检测器在0.8%的像素阈值下，能100%检测目标
  2. Trapped Rate：量化被卡死的严重程度

B. Collision Avoidance Method

STUBBORN使用4个策略避免碰撞：

  1. 暴力破解
  2. 多尺度碰撞障碍图(乐观/悲观碰撞图)
  3. 在探索规划器中做碰撞规避
  4. 始终将访问路径标记为自由空间
