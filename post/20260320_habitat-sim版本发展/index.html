1. Habitat 简介

habitat-sim 是 AI Habitat 提供的一个仿真平台，是一个逼真的高效3D模拟器。Habitat平台提供habitat-sim仿真环境、habitat-lab高层接口、habitat challenge社区挑战三个内容。

1.1 Habitat-sim

Habitat-sim 是一个高性能物理驱动的3D模拟器，支持：

  1. 室内/室外空间的3D 扫描场景(HM3D、MatterPort3D、Gibson、Replica等数据集)
  2. 空间与分段刚性物体的CAD模型(ReplicaCAD、YCB、Google Scanned Objects)
  3. 可配置传感器(RGB\RGB-D\egomotion sensing)
  4. 通过 URDF 描述的机器人(如移动操作机器人 Fetch、固定底座机械臂 Franka、四足机器人 AlienGo)
  5. 刚体力学仿真（通过 Bullet 实现）

在对 Matterport3D 数据集中的场景进行渲染时，Habitat-Sim 在单线程运行下可达到数千帧每秒（FPS），而在单块 GPU 的多进程运行方式下，速度可超过 10,000 FPS。

1.2 Habitat-lab

Habitat-lab是对Habitat-sim的高层封装，包括各种任务的配置，以及一些baselines。

1.3 Habitat-Challenge

Habitat-Challenge是举办的导航相关的比赛，2019年为第一届，举办到2023年。比赛内容为PointGoal \ ObjectGoal \ Rearrange \ OVMM 任务。

2. Habitat-sim 版本的演进

2.1 早期阶段：v0.1.0–v0.1.3

支持对象库导入与程序化场景构建、刚体对象运动学控制、基于 Bullet 的初步刚体物理、WebGL/JavaScript API、PyTorch GPU-GPU 内存共享、以及初步 VR 支持。

2.2 成熟导航与物理阶段：v0.1.4–v0.1.6

强化导航、传感器、物理和数据集支持。

2.3 Habitat 2.0 / 机器人操作跃迁：v0.2.0–v0.2.2

正式向机器人操作与复杂具身任务扩展。

v0.2.0 是一个很关键的转折点：它引入了 ArticulatedObject API，支持 URDF 导入、转动/滑动/球形关节、关节位置/速度/力矩控制与 JointMotor；同时加入了 RobotWrapper、异步渲染、SceneDataset 自动化实例化、分层传感器框架重设计，以及 cube map / fisheye / equirectangular 等新型球面传感器，还有 VHACD、VoxelField、SDF 等能力。

v0.2.1 则继续完善 RobotWrapper、PBR/IBL、WebXR/WebVR 手部功能和场景热切换。

v0.2.2 又加入了音频传感器、Python interactive Viewer、FairMotion 接口、PIP 安装支持、IBL 原型、更多关节/碰撞辅助功能，并对 Python 版本、模块路径和对象操作 API 做了 breaking changes。

2.4 批量渲染与数据/渲染增强阶段：v0.2.3–v0.2.5

偏向渲染基础设施、导航工具完善、数据下载器和 PBR 管线增强。

v0.2.3 的重点包括 Nav 模块改进、NavMesh islands、topdown map、NavMeshSettings JSON 序列化，以及 Batched Renderer prototype；

v0.2.4 进一步升级到 Python 3.9，改进 camera unproject、MultiGoalShortestPath，并大幅推进 batch renderer / gfx replay；

v0.2.5 则强化了 PBR rendering、IBL、NavMeshSettings 重构、Bullet 升级到 3.2.5、Articulated object 的 skinning 与 semantics，同时让 datasets_download.py 支持 Hugging Face 仓库和 HSSD。

2.5 0.3.x 阶段：语义增强、接口整理、工程清理

语义扩展、配置体系增强和代码库清理整合。

v0.3.0 延续了 0.2.5 后的基础，进入 0.3 大版本；

v0.3.1 的亮点是 Semantic Region Support、Panoptic/Instance ID 对 semantic sensor 的支持、PBR shader 下的 skinned mesh 渲染、以及更多 ArticulatedObject 暴露；

v0.3.2 加入 Simulator.reset() 对对象原始状态恢复、raycasting 精度改进、MarkerSets 配置/API、以及 ArticulatedObject 的 AABB 与映射支持；

v0.3.3 则明显偏工程化，加入调试绘制工具、数据下载器更新，并且移除了 JS build、C++ Agent/Controls、过时函数和 datatool，同时把多个 habitat 库合并成单库以消除循环依赖。

3. 总结

habitat-sim的版本演进与habitat-lab是同时进行的，但是有关habitat-sim的文档教程却很少，官方的文档也并非依据新版本编写，而目前很多论文都是基于旧版本进行代码实现的，所以后面很多版本虽然功能更丰富，但是并没有被用到，比如24年INPS顶会的论文还在使用19年的仓库分支，所以入门Habitat还是一个不小的挑战。
