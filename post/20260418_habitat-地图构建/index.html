前言

对于具身智能与目标导航问题，一般避不开地图的构建，已有的建图方法有很多，建图的类型也不少，这里介绍一下常见的建图种类与建图方法

1. 建图种类与建图方法

首先先介绍一下habitat能直接拿到的图类信息，agent的位姿与sensor的位姿自不用说，除此之外，我们还可以直接得到RGB图像、depth图像以及Semantic图像，如下图所示:

RGB图像与depth图像可以直接从HabitatSimRGBSensor与HabitatSimDepthSensor中得到，真实的像素级语义地图也可以直接从HabitatSimSemanticSensor得到，不过一般semantic_sensor的使用是作为GT来用的，一般会通过目标检测与目标分割模型来得到语义信息。

但是，semantic_sensor得到的语义图，存储的实例id信息，要想得到如上图的语义图，需要经过几层映射，首先，用于训练使用的HM3D与MP3D数据集明确将类别归类到了40个类别(还要再外加一个unlabel)，虽然训练集中的目标种类就6类。实际上，数据集中每个场景中出现的种类平均有100多个，这些种类又压缩到40个。

映射关系为 instance_id->instance_name->hm3d_cat_name->hm3d_cat_id，habitat-sim提供了habitat-sim/data/hm3d_semantics/hm3dsem_category_mappings.tsv文件来提供 instance_name->hm3d_cat_name的映射，habitat-lab代码提供了 instance_id->instance_name的映射，hm3d_cat_name->hm3d_cat_id的映射可以自己通过代码定义，以及自定义每个类别的的颜色。

而所谓的建图一般就依赖RGB图像、depth图像、senmantic_info、position and rotation这几个信息。

1.1 建图种类

  • 占据栅格图
    • 占据栅格图是最常见的，一般有三种占据信息 free、unknow、occupide
  • 历史轨迹图
    • 保存历史运行轨迹
  • 语义图
    • 语义图很多都是构建多少个类别构建多少个通道

而论文常见的地图大都是上述三种图融合的结果，比如一张带有颜色、运行轨迹的地图

1.2 建图方法

  1. Active Neural SLAM
    • LEARNING TO EXPLORE USINGACTIVE NEURAL SLAM论文提出了一种主动神经网络建图的方法，输入RGB图像与位姿增量信息，直接通过神经网络生成预测的地图
  2. 传统建图方法
    • Object Goal Navigation using Goal-Oriented Semantic Exploration论文提供一种较为传统的建图方法，深度图与语义信息3D投影，投影到世界坐标系，在一定高度做信息压缩，压缩到2D，得到占据栅格图与语义图
  3. 更为传统的方法
    • PROBABILISTIC ROBOTICS（概率机器人）一书中介绍了更为传统与具有数学基础的建图方法，基于概率的更新

2. 建图实现

2.1 深度点云投影

深度图 + 相机内参 → Open3D create_from_depth_image → 世界坐标系 3D 点云。语义图以 RGB 编码方式与深度图像素对齐，通过 create_from_rgbd_image 同步投影。

2.2 占据栅格更新：Log-Odds 概率模型

不再直接覆写 FREE/OCCUPIED，而是维护一个连续的 log-odds 栅格：

$$ \text{log\_odds} = \ln\frac{p}{1-p} $$

每帧对每个 cell 做贝叶斯增量更新：

$$ L_{t} = L_{t-1} + \underbrace{\ln\frac{p_{\text{free}}}{1-p_{\text{free}}}}_{\text{FREE 证据}} \quad\text{或}\quad L_{t} = L_{t-1} + \underbrace{\ln\frac{p_{\text{occ}}}{1-p_{\text{occ}}}}_{\text{OCC 证据}} $$

导出时用阈值截断：

后验概率 $p$ 离散结果
$p \le 0.35$ FREE
$p \ge 0.75$ OCCUPIED
$0.35 < p < 0.75$ 或未观测 UNKNOWN

阈值带 [0.35, 0.75] 比常见的 [0.50, 0.50] 更保守，单帧证据不足以确认，需要多帧从不同角度观测到同一区域才确认状态，对深度噪声更鲁棒。

2.3 FREE 区域生成：极坐标视锥面填充

旧版用 Bresenham 射线逐线清理 FREE，容易在相邻射线间产生未清理的细缝。新版改用角度桶 + 2D 面填充：

  1. 将当前帧点云投影到 x-z 平面，估计视锥中心方向
  2. 将 360° 离散为 540 个角度桶
  3. 每桶用第 90 百分位距离作为可视远端（$r_{\text{seen}}$），障碍点取最近距离作为遮挡边界（$r_{\text{block}}$）
  4. $r_{\text{free}} = \min(r_{\text{seen}},\ r_{\text{block}} - \text{margin})$
  5. 在视锥 bbox 内做 2D 面填充：距离 ≤ 所在角度桶的 $r_{\text{free}}$ → FREE 证据

2.4 语义地图构建

语义点云通过 2D 投影计数到 semantic[C, H, W] 栅格：

  • 过滤条件：点云高度落入 height_range、语义标签有效
  • 相同 (semantic_id, row, col) 三元组聚合计数，通过 np.unique + return_counts 增量累加
  • 可视化时用 argmax 取得每 cell 的主类别，映射到 41 类 RGB 色表着色
