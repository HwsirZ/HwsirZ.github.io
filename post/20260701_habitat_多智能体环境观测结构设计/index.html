1. 环境封装

在原始多智能体环境的基础上，针对强化学习算法，构建需要用到的观测空间，原始环境 ObjctGoalNavigationRLEnv 根据配置文件，加载环境信息，提供 reset(), step() 等函数，但是这里 reset(), step() 函数返回的只是每个agent自身的RGB观测、depth观测、position等信息，这个时候需要在对这个环境进行一次封装，这个所谓的封装就需要 对这些观测数据进行处理，构造强化学习需要用到的数据格式，方便 runner 进行数据的 collect -> insert -> compute -> train 。

1.1 ObjctGoalNavigationRLEnv 返回的观测数据格式

1.1.1 reset() 返回 → merged_obs 

merged_obs: Dict[str, Dict[str, Any]] = {
    "agent_0": {
        "obs": {                                      # 传感器观测
            "rgb":      np.ndarray, shape (H, W, 3),  dtype uint8,    范围 [0, 255]
            "depth":    np.ndarray, shape (H, W, 1),  dtype float32,   单位 米
            "semantic": np.ndarray, shape (H, W, 1),  dtype int32,     值 [0, 40]（41类，已从 instance id 转换）
        },
        "sensor_states": Any | None,                  # Habitat 内部传感器状态（通常不用）
        "position":  np.ndarray, shape (3,),          dtype float32,   [x, y, z] 世界坐标
        "rotation":  quaternion 对象,                   # Habitat quaternion（有 x, y, z, w 属性）
    },
    "agent_1": { ... },                                # 同上，N 个 agent
}

其中 H, W 由 Habitat YAML 配置决定，默认配置是 480×640。

1.1.2 step() 返回 → (merged_obs, reward, dones, info) 

2.1 merged_obs — 同上，最新帧

2.2 reward

reward: Dict[str, float] = {
    "agent_0": 0.0,                                   # 永远是 0.0
    "agent_1": 0.0,
}

2.3 dones

dones: np.ndarray, shape (N,), dtype float32          # 例: array([0., 0.], dtype=float32)

- 值 0.0 = agent 未 done，1.0 = agent done
- 由 _build_termination() 填充：dones = max(terminated, truncated)

2.4 info

info: Dict[str, Any] = {
    # ======== per-agent 诊断 ========
    "agents": {
        "agent_0": {
            "position":       np.ndarray, shape (3,),  dtype float32,   # [x, y, z] 世界坐标
            "rotation":       quaternion 对象,                            # 四元数朝向
            "goal":           Dict[str, Any] | None,                      # 目标信息
            "distance_to_goal": float | None,                             # 到目标的最短路径距离（米）；无路径时为 None
            "success":        float,                                      # 0.0 或 1.0（stop + 距离 < success_distance）
            "spl":            float,                                      # Success weighted by Path Length [0, 1]
            "soft_spl":       float,                                      # 软 SPL [0, 1]
            "distance_to_goal_reward": float,                             # ★ 上一步距离 - 当前步距离（正=接近目标）
            "is_stop_called": bool,                                       # agent 是否执行了 STOP
            "done":           float,                                      # 0.0 | 1.0
            "terminated":     float,                                      # 0.0 | 1.0  ← habitat episode_over
            "truncated":      float,                                      # 0.0 | 1.0  ← 达到 max_episode_steps
            "collided":       float,                                      # 0.0 | 1.0  ← 前进距离 < forward_step_size × 0.9
            "last_action":    int | None,                                 # 上一步动作 ID 或 None（reset 后第一步）
        },
        "agent_1": { ... },                                                # 同上
    },

    # ======== 全局信息 ========
    "episode_over":       bool,                                            # 是否任何 agent done
    "termination_reason": str | None,                                      # "success_stop" | "failed_stop" | "time_limit" | None
    "dones":              np.ndarray, shape (N,), dtype float32,           # per-agent dones 数组
    "terminated":         np.ndarray, shape (N,), dtype float32,           # per-agent terminated 数组
    "truncated":          np.ndarray, shape (N,), dtype float32,           # per-agent truncated 数组
    "collided":           np.ndarray, shape (N,), dtype float32,           # per-agent collided 数组
    "step":               int,                                             # 当前 episode 步数
    "success":            bool,                                            # 任何 agent success == 1.0
    "metrics":            Any,                                             # Habitat 官方 metrics（通常包含 top_down_map 等）
}

1.1.3 get_info() 返回 — 同上 info

1.1.4 info[“agents”][aid][“goal”] 格式 

goal: Dict[str, Any] = {
    "object_category": str | None,     # 目标物体类别名，如 "chair"
    "num_goals":       int,            # 目标候选位置数量
    "goals": [                         # 目标候选列表
        {
            "position":        list,   # [x, y, z] 世界坐标
            "radius":          float,  # 目标半径
            "object_id":       str,    # 物体 ID
            "object_category": str,    # 物体类别名
            "num_view_points": int,    # view points 数量
        },
        ...
    ]
}

2. 构建观测

参考 maans MultiHabitatEnv 观测空间设计 来源：maans/onpolicy/envs/habitat/Habitat_Env.py

2.1 Space 结构 

self.observation_space       = [gym.spaces.Dict({...})] × N   # Actor 观测，Dict 类型
self.share_observation_space = [gym.spaces.Dict({...})] × N   # Critic 观测，Actor + gt_map
self.action_space            = [gym.spaces.Discrete / Box] × N # 动作

2.2 尺寸变量

变量 含义
N agent 数量
S grid_size 网格尺寸
H 图像高度，取决于 use_resnet 和 use_local 等开关

2.3 Actor 观测 Dict 全通道

地图类

key 条件 shape dtype 内容
global_obs use_single=True (4, H, H) uint8 单 agent 4 通道地图：障碍 / 已探索 / 当前位置 / 历史轨迹
global_merge_obs use_single=False (4|8, H, H) uint8 多 agent 融合地图
local_merge_obs use_local + use_seperated_cnn (4, H_loc, H_loc) uint8 局部分离 CNN 的局部通道

目标类

key 条件 shape dtype 内容
global_goal use_single + use_goal (2, H, H) uint8 单 agent 目标位置 + 历史轨迹
global_merge_goal use_merge_goal (2, H, H) uint8 融合目标图（所有 agent goal + 历史）

轨迹类

key 条件 shape dtype 内容
trace_image use_single_agent_trace (N|1, H, H) uint8 agent 历史经过位置累积图

网格类

key 条件 shape dtype 内容
grid_pos grid_pos (2|2N, S, S) int 位置偏移：栅格坐标 - agent 当前栅格坐标
grid_goal grid_last_goal (2|2N, S, S) int 上一轮 global policy 输出的 goal 栅格
action_mask_obs action_mask (1, S, S) int 可通行掩码
action_mask action_mask (1, S, S) int 同上副本
grid_agent_id grid_agent_id (1, S, S) int 每栅格的 agent ID

朝向/ID 向量类

key 条件 shape dtype 内容
global_orientation use_orientation (1,) long 自身朝向离散化（0-71，5°/bin）
other_global_orientation use_orientation (N-1,) long 其他 agent 相对朝向
vector use_vector_agent_id (N,) float agent ID 的 one-hot 向量

Agent ID 图像类

key 条件 shape dtype 内容
vector_cnn use_cnn_agent_id (1|2|N+1, H, H) uint8 agent ID 的 CNN 通道图

堆叠观测

key 条件 shape dtype 内容
stack_obs 始终存在 (C_total × N, H, H) uint8 所有 agent 全部单 agent 通道直接按通道拼接

2.4 Critic 观测 share_observation_space

= Actor 全部观测 + 以下额外 key（仅在 CTDE 模式下）：

key 条件 shape dtype 内容
gt_map use_centralized_V=True (1, H, H) uint8 Habitat navmesh 真值可通行地图
