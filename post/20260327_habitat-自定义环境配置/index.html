1. habitat 各个模块介绍

对于habitat的环境而言，habitat有4部分组成，4个部分整合到一起就是habitat的环境Env，4个部分分别是:

Scene Dataset
   ↓
Episode Dataset
   ↓
Simulator
   ├── Agents
   ├── Sensors
   └── Actions
   ↓
Task
   ├── Goal definition
   ├── Success condition
   ├── Reward / Metrics
   └── Episode termination
   ↓
Env

查看yaml配置文件可知，habitat是顶级字段，这个字段下，还有一些环境信息，还有几个字段的配置就是 simulator,task,dataset，scene data 也就是场景路径的配置在 simulator下。

Scene Data 是场景文件，有时候还需要场景对应的语义文件，episodes dataset 是episodes数据，定义一个episode的目标类别，agent的起点、朝向，scene_id，simulator是对仿真环境的定义，如agent的信息，agent的传感器信息，以及agent的动作空间，task是对任务的定义，比如目标导航任务，需要定义成功的条件，成功后的奖励，以及对成功的评价指标，episode结束的判断等信息。上述4部分拼接到一起就是Env。

理论上，并不需要yaml配置文件去设置，可以自己定义各部分的内容，然后用于环境的初始化，官方给的yaml配置都是针对特定任务的，很难适应自己的任务类型，所以构建自己的环境很有必要。

Note

habitat-lab封装的Env以及RLEnv只针对单智能体，要想支持多智能体，需要自己更改 Simulator、Datasets、Task的相关代码，也就是Simulator中的多智能体配置，为每个agent设置其传感器，为每个agent设置初始的位姿(因为episode datasets只支持单智能体的)，读取 episode datesets，设置多智能体的Task，也就是设置episode结束的标志，任务成功的标志，此外还可以自定义评测，也就是自动评测，什么成功率之类的。

2. 各子部分详解

2.1 scene Data

habitat其实支持很高的自定义场景，以及agent的定制，这里scene Data直接使用 HM3D_v0.2 数据集，这里直接指明scene的数据路径就行了。

habitat 加载环境慢的问题

首次创建环境时，环境的加载很慢，这里并不是真的在加载环境，环境非常大，一次性加载完也不可能，首次加载慢的原因是，habitat似乎是直接加载所有的episode data，正是因为如此，环境的建在才如此之慢，这里测试时可以限制使用的场景，不加载所有场景的eisodes，只加载一个的，就会快很多。

2.2 Episode Dataset

对于episode数据集，这个其实挺坑的，habitat的任务数据集，在数据集里面，也就是json文件中，还会加载一次场景路径，这个时候场景路径需要在运行路径下，负责会找不到，这个时候可以使用软链接。

查看habitat的Env 类的代码可知，参数的传递有cfg，也就是加载的yaml的内容，除此之外还有一个就是dataset，这个dataset就是episode datasets。如下图所示，habitat根据task名调用make_dataset直接加载对应的episode数据，这个类会返回一个ObjectNavDatasetV1对象，如果你使用的是objectNav的episode dataset的话
